The users stories we completed during this sprint include:

  1. As a user, I can access the website.
  2. As a user, I can visit the url at ‘/’ and see our homepage, which has our team logo, an unordered list of events, and a link to new events.
  3. As a user, I can visit your API at ‘/api/events’ and retrieve an object with key events containing all events in JSON format.
  4. As a user, I can search for events by submitting a GET request
  5. As a user, I can visit the url at /about and see information about our company
  6. As a user, I can visit a page for creating new events, which contains a form that is validated

Since our sprint team meeting, we have worked to create a functional website for the user. We have created the homepage, an About Us page, an Events page, and more for the user to browse and have allowed the user to access and search our API for events. We initially encountered a roadblock in getting the environment setup, but with some help from Kyle everything was functional pretty quickly. Apart from the initial setup, one of our main challenges during this sprint was figuring out git and how to branch/merge properly but that was quickly sorted out with some googling and trial and error. We worked through our troubles pretty seamlessly because the team had good communication through Facebook chat. Everyone was up-to-date on what we were doing and everyone was updating the Waffle.io board properly which helped facilitate a smooth workflow. We are currently passing 37 of the 41 tests at the end of the first sprint.
